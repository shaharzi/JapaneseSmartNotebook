# מחברת יפנית חכמה

אפליקציית לימוד יפנית לדוברי עברית, עם חיפוש מילים, אוצר מילים אישי וניתוח AI של מילים ומשפטים.

## יכולות עיקריות

- חיפוש ולימוד מילים ביפנית מתוך מאגר מובנה, כולל היראגנה, רומאג'י והשמעה קולית
- ניתוח AI של מילה, ביטוי או משפט בעברית או ביפנית: תרגום, פירוק דקדוקי, הסבר חלקיקים ומשפט לדוגמה
- שמירת תוצאות AI לאוצר המילים האישי
- הוספה, עריכה ומחיקה ידנית של מילים
- כל הנתונים האישיים נשמרים מקומית בדפדפן (LocalStorage) ואינם נשלחים לשרת חיצוני מלבד בקשת הניתוח עצמה
- מטמון מקומי לתוצאות AI שכבר חושבו, תצוגת התקדמות בזמן ניתוח, וכפתור ביטול
- כפתור "בדיקת חיבור" לאימות שמפתח ה-API והמודל מוגדרים כראוי

## דרישות

- .NET 10 SDK
- מפתח OpenAI API פעיל (עם יתרת חיוב בפלטפורמת ה-API, בנפרד ממנוי ChatGPT)

## הפעלה

### Windows (PowerShell)

```
run.ps1
```

### Windows (Command Prompt)

```
run.cmd
```

שתי הסקריפטים מבקשים הדבקת OpenAI API key, מגדירים ברירת מחדל למודל (`gpt-5-mini`) אם `OPENAI_MODEL` לא הוגדר, ומריצים `dotnet run`. המפתח אינו נשמר לקובץ.

לאחר ההפעלה, פתחו את הכתובת שמוצגת במסוף. במסך "ניתוח AI", מומלץ ללחוץ תחילה על "בדיקת חיבור".

## חשוב

מנוי ChatGPT ומערכת החיוב של OpenAI API נפרדים. מפתח יכול להיות תקין אך ללא יתרת API או ללא אמצעי תשלום בפלטפורמת ה-API.

## משתני סביבה

| שם | חובה | ברירת מחדל |
|---|---|---|
| `OPENAI_API_KEY` | כן | אין |
| `OPENAI_MODEL` | לא | `gpt-5-mini` |

## מבנה הפרויקט

- `Program.cs` — שרת ASP.NET Core Minimal API: משרת קבצים סטטיים ומבצע קריאות ל-OpenAI מהשרת בלבד (המפתח אינו נחשף לדפדפן)
- `wwwroot/index.html` — כל הממשק: חיפוש, אוצר מילים, טופס הוספה ידנית ומסך ניתוח AI
- `run.ps1` / `run.cmd` — סקריפטים להפעלה מקומית ב-Windows
